# DIA-HMD Bug Fixes (2026-04-06)

Six bugs were found by testing the pipeline as a fresh user on three environments (WSL2 local, NIG SLURM, SHIROKANE SGE). All six are now fixed.

---

## Bug 1: Config restore race condition (CRITICAL)

**What was broken:** `run.sh` would edit `config.sh` with the user's paths, submit scheduler jobs, then *immediately restore* the original `config.sh`. By the time the scheduler jobs actually started running (seconds to minutes later), `config.sh` was back to placeholder values like `/path/to/diann-2.0.2.img`.

**Fix:** Instead of editing `config.sh` in place, `run.sh` now writes a separate `config.run.sh` file with the user's values. `config.sh` sources it if it exists. No more backup/restore dance.

**Files changed:** `run.sh`, `scripts/config.sh`, `.gitignore`

> **ADHD version:** Imagine you write your friend's address on a sticky note, hand it to a courier, then immediately erase the sticky note. The courier looks at the note and sees... nothing. That's what was happening. The fix: write the address on a *separate* piece of paper that doesn't get erased.

---

## Bug 2: `--diann` relative path not resolved (BLOCKER)

**What was broken:** `run.sh` converts `--input`, `--fasta`, and `--proteome` to absolute paths, but forgot to do the same for `--diann`. Since the pipeline then does `cd scripts/` before running, a relative path like `diann-2.0.2.img` points to `scripts/diann-2.0.2.img` -- which doesn't exist. Every README example used a relative path.

**Fix:** Added path resolution for `--diann` when it points to an actual file on disk (skipped for Docker image names like `biocontainers/diann:2.0.2`).

**Files changed:** `run.sh`

> **ADHD version:** You're in your living room and say "the book is on the table." Then you walk to the kitchen and look at the table there -- wrong table. The fix: say "the book is at 123 Main St, living room, coffee table" -- an absolute address that works from anywhere.

---

## Bug 3: Local fallback swallows failures (BLOCKER)

**What was broken:** When running without a scheduler (local machine), if DIA-NN failed, the pipeline kept going anyway. The `submit_job` function piped output through `tee`, which masked the exit code, so every step "succeeded" even when it actually crashed. Downstream steps then failed with confusing cascading errors.

**Fix:** After `tee`, check `PIPESTATUS[0]` (the real exit code of the script). If non-zero, print an error and stop immediately.

**Files changed:** `scripts/config.sh`

> **ADHD version:** A teacher grades a test. The student got an F, but the teacher's pen only writes "PASS." Every subsequent teacher sees "PASS" and waves the student through. The fix: actually check the score before stamping "PASS."

---

## Bug 4: Scheduler jobs can't find apptainer (BLOCKER)

**What was broken:** Scheduler jobs (SGE/SLURM) start with a fresh, empty environment. On HPC clusters where `apptainer` requires `module load apptainer`, the jobs couldn't find it. The auto-detection fell through to `docker` (which was actually `podman`), which also failed.

**Fix:** Two parts: (1) `run.sh` now detects the user's `MODULEPATH` and writes it to `config.run.sh` so scheduler jobs inherit it. (2) `run_container()` now calls `ensure_tool apptainer` before the auto-detection chain, giving the module system a chance to load apptainer.

**Files changed:** `run.sh`, `scripts/config.sh`

> **ADHD version:** You know where the hammer is in *your* workshop. You hire a contractor who shows up to a completely empty workshop and says "I don't have a hammer, but I found these pliers, let me try those." The fix: leave a note in the workshop saying "hammer is in aisle 3, shelf 2" so anyone can find it.

---

## Bug 5: DIA-NN memory estimate too low (BLOCKER)

**What was broken:** The pipeline allocated 4 GB per thread for DIA-NN. With 4 threads = 16 GB total. DIA-NN actually needed 16.7 GB (loading a 13 GB `.raw.dia` file), so the scheduler killed the job for exceeding its memory limit.

**Fix:** Increased from 4 GB to 6 GB per thread (4 threads = 24 GB). Updated the README resource table to match.

**Files changed:** `scripts/Complete_pipeline.sh`, `README.md`

> **ADHD version:** You book a rental car with a 50-liter tank for a trip that needs 52 liters. You run out of gas 2 km from the destination. The fix: book the car with the 70-liter tank -- give yourself headroom.

---

## Bug 6: Results copied before scheduler jobs finish (MINOR)

**What was broken:** After submitting scheduler jobs, `run.sh` immediately tried to copy results from `Peptidomics_Results/`. But scheduler jobs haven't run yet -- they're just queued. The directory doesn't exist, so nothing gets copied. The user never gets their results in `results/`.

**Fix:** When a scheduler is detected, print a message telling the user where results will appear and the command to copy them after jobs finish. On local (no scheduler), copy immediately as before.

**Files changed:** `run.sh`

> **ADHD version:** You order a pizza, then immediately open the front door to grab it. It's not there yet -- the delivery driver just got the order. The fix: wait for the doorbell, *then* open the door. (Or in this case, tell the user "check the door in 30 minutes.")
